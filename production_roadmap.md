## Phase 2 — Security Hardening
### *Stop the bleeding — protect your API keys, user data, and infrastructure*
**Target: Ship before any public launch or sharing**

---

### 2.1 · Authenticate `/api/ai-model`
**Complexity:** 🟡 S — **Priority:** P0 🔴

Currently: Zero authentication. Anyone on the internet can POST to your OpenRouter proxy endpoint and consume your API budget.

**Fix:** Add `currentUser()` check at the top. Return 401 if no session. Optionally validate the `model` param against `AI_MODELS` allowlist.

```typescript
// Add at top of route handler:
const user = await currentUser();
if (!user) return new Response("Unauthorized", { status: 401 });

const allowedModels = AI_MODELS.map(m => m.id);
if (!allowedModels.includes(model)) {
  return new Response("Invalid model", { status: 400 });
}
```

```
Impact: Prevents API key theft, cost explosion, abuse
Files: app/api/ai-model/route.ts
```

---

### 2.2 · Authenticate `/api/chats`
**Complexity:** 🟡 S — **Priority:** P0 🔴

Currently: Zero authentication. Any request with a `frameId` (0–9999 integers) can overwrite any user's chat history. A script can enumerate all 10,000 IDs in under a minute.

**Fix:** Add `currentUser()` check. Verify the frameId belongs to a project owned by the user before updating.

```
Impact: Prevents data tampering and data loss for all users
Files: app/api/chats/route.ts
```

---

### 2.3 · Authenticate `/api/imagekit-upload`
**Complexity:** 🟡 S — **Priority:** P0 🔴

Currently: Zero authentication. Anyone can upload arbitrary files to your ImageKit account. Can be used for phishing, malware hosting, CSAM — all billed to your account.

**Fix:** Add `currentUser()` check. Return 401 if no session. Optionally add file type server-side validation.

```
Impact: Prevents unauthorized storage abuse, legal liability
Files: app/api/imagekit-upload/route.ts
```

### 2.4 · Add Rate Limiting on All Expensive Endpoints
**Complexity:** 🟠 M — **Priority:** P0

No endpoint has rate limiting. A single user or bot can:
- Create unlimited projects (bypassing credit system entirely via script)
- Send unlimited AI requests per second
- Upload unlimited images

**Fix:** Use **Upstash Redis + `@upstash/ratelimit`**.

```typescript
// Example config per endpoint:
// /api/ai-model: 10 requests / 60 seconds per user
// /api/projects: 5 requests / 60 seconds per user  
// /api/imagekit-upload: 20 requests / 60 seconds per user
```

Add a `lib/rate-limit.ts` helper and wrap each route with `ratelimit.limit(identifier)`.

```
Install: @upstash/ratelimit @upstash/redis
Files: lib/rate-limit.ts + all api/*/route.ts files
```

### 2.5 · Move `projectId` and `frameId` Generation Server-Side
**Complexity:** 🟡 S — **Priority:** P1

Currently, the client generates `projectId` (UUID) and `frameId` (random int) and sends them to the server. A malicious user could:
- Overwrite an existing project by sending a known `projectId`
- Craft a collision attack on `frameId`

**Fix:** Generate both IDs server-side in `/api/projects`. Return them in the response. Client uses the returned IDs for redirect.

```
Impact: Eliminates client-controlled primary key injection
Files: app/api/projects/route.ts, app/_components/Hero.tsx
```

## Phase 3 — Performance
### *Make it fast enough to not lose users*
**Target: Ship within first week post-launch**

### 3.1 · Fix N+1 Query in Project List
**Complexity:** 🟡 S — **Priority:** P0

`/api/project_list` executes 1 query for projects + 1 per project for frames + 1 for chats. For 20 projects: 41+ sequential DB round-trips per page load.

**Fix:** Replace with a single JOIN query using Drizzle's `leftJoin` to fetch projects + frames + chats in one query, then reshape the result in JS.

```
Impact: Page load drops from ~500ms to ~50ms for project list
Files: app/api/project_list/route.ts
```

---

### 3.2 · Add Database Indexes
**Complexity:** 🟡 S — **Priority:** P0

No indexes exist on any FK or lookup column. Every query does a full table scan.

**Fix:** Add indexes via Drizzle schema:

```typescript
// config/schema.ts
export const projectTable = pgTable("projects", { ... }, (t) => ({
  createdByIdx: index("project_created_by_idx").on(t.createdBy),
}));

export const frameTable = pgTable("frames", { ... }, (t) => ({
  projectIdIdx: index("frame_project_id_idx").on(t.projectId),
}));

export const chatTable = pgTable("chats", { ... }, (t) => ({
  frameIdIdx: index("chat_frame_id_idx").on(t.frameId),
}));
```

Then run `drizzle-kit push` to apply.

```
Impact: Query time drops from O(n) to O(log n) as data grows
Files: config/schema.ts + migration
```

### 5.3 · Add Context Window Management (Message Summarization)

### 6.5 · Add Background Job Queue for AI Generation

4. A background worker processes the job and saves the result.
